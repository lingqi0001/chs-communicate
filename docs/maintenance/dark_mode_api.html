1 The Initialization Logic

We use window.onload to ensure the script only runs after your page has fully loaded all elements. This prevents errors during startup.

Universal Logic Template 
<script>
    // This function waits for your page to be 100% ready
    window.onload = () => {
        
        // Ask the bridge: Is the main app in Dark Mode?
        if (AppBridge.isDark()) {
            // [A] Your code to ACTIVATE Dark Mode goes here
            console.log("Setting UI to Dark...");
        } else {
            // [B] Your code to ACTIVATE Light Mode goes here
            console.log("Setting UI to Light...");
        }

    };
</script>

Core Principles

  • Passive Mode: The SDK only checks once at startup. You retain 100% control after initialization.
  • Zero Config: No complex IDs or tokens required. Just include the script and use it.

Pro Tip

If you are using Tailwind CSS, your Dark Mode logic would simply be:
document.documentElement.classList.add('dark');
