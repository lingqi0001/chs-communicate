Technical Documentation

Gallery API

Summon native full-screen touch gallery for multi-image swiping and original ratio downloading.

Part 1: Extension Developers

! Step 0: Parent Bridge Connection

Note: Extensions must include the SDK Bridge script before calling parent gallery components.

<script src="../js/app-bridge.js"></script>

1 Standard Code Pattern

Extensions must call AppBridge.openGallery(images, index) to summon native gallery overlays. Accepts single URL strings or string arrays. 

<script>
    // Example 1: Open single image view
    function viewSingleImage() {
        const imageUrl = "https://example.com/assets/poster.jpg";
        AppBridge.openGallery(imageUrl);
    }

    // Example 2: Open multiple images (active starting at second image)
    function viewGroupGallery() {
        const images = [
            "https://example.com/img1.jpg",
            "https://example.com/img2.jpg",
            "https://example.com/img3.jpg"
        ];
        AppBridge.openGallery(images, 1);
    }
</script>
Part 2: Platform Core Reference

1 AppModules.Gallery.open

Syntax
AppModules.Gallery.open(encodedImages, startIdx*);

Parameters

  • `encodedImages`: String - URL-encoded JSON string of the images array (encodeURIComponent(JSON.stringify(imagesArray))).
  • `startIdx`: Number (Optional) - Initial active image index upon opening, defaults to 0.

Under The Hood
The platform locks scrolling, ignores standard pull-to-refresh behaviors to prevent web page conflict, executes damping open-scale transition logic, and positions the modal on a high-priority viewport layer (z-[3000]).

2 Dynamic Gesture Engine

Interaction bindings inside the gallery viewport are dynamically registered during the platform initialization init() phase, maintaining perfect decoupling with zero inline DOM listeners:

  • Touch Swipe Gestures: Supports single-finger horizontal swipe tracking. Drag distances exceeding 60px trigger visual flip transitions.
  • Keyboard Navigation: Physical desktop keys such as ArrowLeft, ArrowRight, and Escape automatically map to prev/next slide movements and close events.
  • Zero-Jitter Preloading Cache: Dynamically instantiates parallel browser preloading for index - 1 and index + 1 image paths to ensure seamless visual switches.

Parameter Glossary

  • `images`: Image source. Supports external URLs, local paths, or Base64 data strings.
  • `* (Asterisk)`: Indicates an Optional parameter.
  • `AppBridge.openGallery`: Encapsulates URL formatting and JSON encoding. Highly recommended over direct core invocations.

Core Best Practices

  • 1. Type Consistency: Although a string single URL is parsed gracefully, passing arrays is recommended for scalable image collections.
  • 2. Scroll Release: Scroll lock triggers are cleaned up automatically upon gallery exit, restoring scroll overflow to parent documents.
