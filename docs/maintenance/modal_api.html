AppModules.Modal Developer Guide

All methods are Async and must be preceded by `await`.

1 Alert (Notification)

Syntax
await AppModules.Modal.alert(title, body, btnText*);

Returns
true (when user clicks the button).

Usage
Inform users of a result. No branching logic required.

2 Confirm (Binary Choice)

Syntax
const result = await AppModules.Modal.confirm(title, body, confirmText*, cancelText*);

Returns
Boolean (true for Confirm, false for Cancel/Backdrop).

Logic
if (result) { /* proceed */ }.

3 Prompt (Single Input)

Syntax
const input = await AppModules.Modal.prompt(title, body, defaultValue*);

Returns

  • String: The text entered by the user (including empty string "").
  • null: If the user clicked "Cancel".

Critical Note

Always check for `null` to ensure the user didn't cancel the action.

if (input === null) return; // Stop execution if cancelled

4 ShowCustom (Advanced/Complex UI)

Syntax
const result = await AppModules.Modal.showCustom(title, html, buttons*);

Button Schema

const buttons = [
    { 
        text: "Display_Name", // String: Button label
        value: "Any_Value",   // Mixed: Data returned upon click
        primary: true         // Boolean: Bold(true) or Normal(false) styling
    },
    // ... supports multiple button objects
];

const result = await AppModules.Modal.showCustom(title, html, buttons);

Returns
The value defined in the clicked button object.

Parameter Glossary

  • `title`: Header text of the modal.
  • `body / html`: Content area. Supports standard HTML tags for formatting.
  • `* (Asterisk)`: Indicates an Optional parameter with a default fallback.
  • `defaultValue`: Pre-filled text inside the Prompt input field.

Best Practices

  • 1. Unique IDs: When using `showCustom`, ensure HTML elements have unique IDs to correctly retrieve data via `document.getElementById`.
  • 2. Auto-Cleanup: The system handles scroll locking and event listener cleanup automatically. No manual DOM manipulation is required after the Promise resolves.
