Technical Documentation

Alert & Prompt API

All methods are Async and must be preceded by `await`.

Part 1: Extension Developers

! Step 0: The Essential Bridge

Note: This script must be included before calling any SDK methods.

<script src="../js/app-bridge.js"></script>

1 Standard Code Pattern

Extensions must use await AppBridge to trigger platform UI components. 

<script>
    async function example() {
        // Simple notification
        await AppBridge.alert("Success", "Task complete.");

        // Binary Choice
        const ok = await AppBridge.confirm("Delete?", "Action is permanent.");
        if (ok) { /* handle delete */ }
        
        // Data Input
        const val = await AppBridge.prompt("Set Nickname", "Enter below:", "Default Name");
    }
</script>
Part 2: Technical Reference

1 Alert (Notification)

Syntax
await AppModules.Modal.alert(title, body, btnText*);

Returns
true (when user clicks the button).

Usage
Inform users of a result. No branching logic required.

2 Confirm (Binary Choice)

Syntax
const result = await AppModules.Modal.confirm(title, body, confirmText*, cancelText*);

Returns
Boolean (true for Confirm, false for Cancel/Backdrop).

Logic
if (result) { /* proceed */ }.

3 Prompt (Single Input)

Syntax
const input = await AppModules.Modal.prompt(title, body, defaultValue*);

Returns

  • String: The text entered by the user.
  • null: If the user clicked "Cancel".

Critical Note

Always check for `null` to ensure the user didn't cancel.

if (input === null) return; // Stop execution

4 ShowCustom (Complex UI)

Syntax
const result = await AppModules.Modal.showCustom(title, html, buttons*);

Button Schema

const buttons = [
    { 
        text: "Display_Name", // String: Button label
        value: "Any_Value",   // Mixed: Data returned upon click
        primary: true         // Boolean: Bold(true) or Normal(false) styling
    }
];

Returns
The value defined in the clicked button object.

Parameter Glossary

  • `title`: Header text of the modal.
  • `body / html`: Content area. Supports HTML tags.
  • `* (Asterisk)`: Indicates an Optional parameter.
  • `defaultValue`: Pre-filled text inside Prompt.

Best Practices

  • 1. Unique IDs: Ensure unique IDs for input retrieval.
  • 2. Auto-Cleanup: System handles all scroll locking automatically.
